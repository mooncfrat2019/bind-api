# Методика нагрузочного тестирования BIND API

## 1. Цель тестирования

Определить производительность API при добавлении A-записей в DNS-зону.

## 2. Среда тестирования

### 2.1. Аппаратное обеспечение

| Компонент | Требование |
|-----------|-----------|
| CPU | 2+ ядра |
| RAM | 4+ GB |
| Диск | SSD |
| ОС | РедОС 7.3 |

Это нагрузочное тестирование можно запустить в docker, в качестве образа как раз используется РедОС 7.3 (redos7c/ubi:latest)

### 2.2. Программное обеспечение

- Docker 20.10+
- Docker Compose 2.0+
- Git

## 3. Параметры теста

| Параметр | Значение |
|----------|----------|
| Количество зон | 1 |
| Имя зоны | `loadtest-a.local` |
| Количество A-записей | 2000 |
| Скорость запросов (RPS) | 20 |
| Длительность теста | 120 секунд |

## 4. Запуск теста

### 4.1. Подготовка

```bash
git clone https://github.com/mooncfrat2019/bind-api.git
cd bind-api/test/load
```

### 4.2. Запуск контейнеров

```bash
docker-compose up -d
```

### 4.3. Настройка API-ключа

```bash
make setup
```

### 4.4. Запуск нагрузочного теста

```bash
make test
```

### 4.5. Остановка контейнеров

```bash
make down
```

## 5. Критерии успеха

| Показатель | Критерий |
|------------|----------|
| Успешность запросов | 100% (код 200) |
| Средняя задержка | менее 100 мс |
| p95 задержка | менее 200 мс |
| Ошибки в логах | отсутствуют |

## 6. Сбор результатов

Результаты сохраняются в директории `results/YYYYMMDD_HHMMSS/`:

- `add_records.bin` - сырые данные vegeta
- `add_records_report.json` - отчёт в JSON
- `add_records_report.txt` - текстовый отчёт

## 7. Анализ результатов

Пример успешного выполнения:

```

```

## 8. Очистка

```bash
make clean
```
